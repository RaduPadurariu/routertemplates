HTML colors

    rgb(255, 0, 0) is red,

    Hue is a degree on the color. The values can be from 0 to 360 where
    0 is red,
    120 is green,
    240 is blue,
    Saturation is a percentage value.
    0% is a shade of grey,
    100% is full color,
    Lightness show the percentage of light where:
    0% is black,

Emojis are characters from UTF-8 character set: 😄 😍 💗. ASCII was the first character encoding standard. It defines 128 different characters: numbers(0-9), English letters (A-Z) and other special symbols.
UTF-8 is identical to ASCII for the values from 0 to 127, does not use values from 128 to 159, is identical to ANSI and 8859-1 for values between 160-255. UTF-8 continues from value 256 with more than 10.000 different characters.

xHTML vs HTML

XHTML is a stricter, more XML-based version of HTML. XML is a markup language where all documents must be marked up correctly.
The most important differences from HTML are:

  • <!DOCTYPE> is mandatory
  • The xmlns attribute in <html> is mandatory
  • <html>, <head>, <title> and <body> are mandatory
  • Elements must always be properly nested
  • Elements must always be closed
  • Elements must always be in lowercase
  • Attribute name must always be in lowercase
  • Attribute values must always be quoted
  • Attribute minimization is forbidden

HTML Style guide

A consistent, clean, and tidy HTML code makes it easier for others to read and understand your code. Those are the major guides:

  • Always declare document type: <DOCTYPE html>
  • Use lowercase elements and attributes names: <div style="" > </div>
  • Close all HTML elements: <p> </p>, <img />
  • Always quote attribute values: <table class="striped">
  • Always specify alt, width and height for images.
  • Don't use space after equal: This is bad: <link rel = "stylesheet">
  • Avoid long code lines.
  • Do not add blank lines, spaces, or indentations without a reason.
  • Add the lang attribute: <html lang="en-us">
  • Use comments in your code.

HTML Graphics and Media

The <canvas> element is use to draw graphics via Java Script. Don't forget to set width and height for it.
The <svg> element is a container for SVG graphics (Scalable Vector Graphics). Is the most common used.

Multimedia on the web is sound, music, videos, movies, and animations. Multimedia comes in many different formats.

HTML Forms

An HTML <form> element is a container for different types of input elements and is used to collect data from the user.
An <input> element can be displayed in many ways, depending on the type attribute.

Form attributes

  • name - Specifies the name of the form.
  • action - Specifies where to send the form-data when a form is submitted.
  • method - Specifies the HTTP method to use when sending form-data.
  • autocomplete - Specifies whether a form should have autocomplete on or off.
  • enctype - Specifies how the form-data should be encoded when submitting it to the server (only for method="post").
  • novalidate - Specifies that the form should not be validated when submitted.
  • rel - Specifies the relationship between a linked resource and the current document.
  • target - Specifies where to display the response that is received after submitting the form.
  • accept-charset - Specifies the character encodings used for form submission.

Form elements

  • <form> - Defines an HTML form for user input.
  • <input> - Defines an input control.
  • <textarea> - Defines a multiline input control (text area).
  • <label> - Defines a label for an input element.
  • <fieldset> - Groups related elements in a form.
  • <legend> - Defines a caption for a fieldset element.
  • <select> - Defines a drop-down list.
  • <optgroup> - Defines a group of related options in a drop-down list.
  • <option> - Defines an option in a drop-down list.
  • <button> - Defines a clickable button.
  • <datalist> - Specifies a list of pre-defined options for input controls.
  • <output> - Defines the result of a calculation.

Examples of attributes for form elements:

  • <label> with for - for should be equal to id attribute of the input. Ex: <label for="fname">First name:</label>
  • <select> with size - size specify the number of visible values.
  • <select> with multiple - multiple attribute allow the user to select more than one value with CTRL key.
  • <textarea> with rows and cols - rows specify the visible numbers of lines in a text area and cols specify the visible width of a text area.
  • <input> with list and <datalist> with id - The list attribute of input must refer to the id attribute of datalist element.
  • <output> with name and for - name must be equal with name.value from form attribute called oninput, for must contain the id values of inputs.

Input types

  • <input type="text"> - defines a single-line text input field.
  • <input type="password"> - defines a password field.
  • <input type="submit"> - defines a button for submitting form data to a form-handler. The form-handler is specified in the form's action attribute.
  • <input type="reset"> - defines a reset button that will reset all form values to their default values.
  • <input type="radio"> - defines a radio button (select only one).
  • <input type="checkbox"> - define a checkbox (multiple selections).
  • <input type="button"> - defines a button. onclick attribute is used to acces the JS function.
  • <input type="color"> - is used for input fields that should contain a color.
  • <input type="date"> - is used for input fields that should contain a date. With attributes min and max we can add restrictions to dates.
  • <input type="datetime-local"> - specifies a date and time input field, with no time zone.
  • <input type="email"> - is used for inputs that should contain an e-mail address.
  • <input type="image"> - defines an image as a submit button.
  • <input type="file"> - defines a file-select field and a "Browse" button for file uploads.
  • <input type="hidden"> - defines a hidden input field (not visible to user).
  • <input type="month"> - allows the user to select a month and year.
  • <input type="number"> - defines a numeric input field. Can use min and max attributes to add restrictions.
  • <input type="range"> - defines a control for entering a number whose exact value is not important. Can use min and max to add restrictions.
  • <input type="search"> - is used for search fields.
  • <input type="tel"> - is used for inputs fields that should contain a telephone number. Can use patter attribute to set the desire configuration of the number.
  • <input type="time"> - allows the user to select a time (no time zone).
  • <input type="url"> - is used for inputs fields that should contain a URL address.
  • <input type="week"> - allows the user to select a week and year.

Input Restrictions

  • value - Specifies the default value for an input field.
  • checked - Specifies that an input field should be pre-selected when the page loads (for type="checkbox" or type="radio").
  • disabled - Specifies that an input field should be disabled.
  • required - Specifies that an input field is required (must be filled out).
  • min - Specifies the minimum value for an input field.
  • max - Specifies the maximum value for an input field.
  • maxlength - Specifies the maximum number of character for an input field.
  • size - Specifies the width (in characters) of an input field.
  • pattern - Specifies a regular expression to check the input value against.
  • readonly - Specifies that an input field is read only (cannot be changed).
  • step - Specifies the legal number intervals for an input field.

Input attributes

  • multiple - specifies that theuser is allowed to enter more than one value in an input field.
  • placeholder - specifies a short hint that describes the expected value of an input field.
  • autofocus - specifies that an input field should automatically get focus when page loads.
  • width and height - specify the size of an input field.
  • list - reffers to <datalist> element that contain a pre-defined options for an input element.
  • autocomplete - specifies whether a form or an input field should have autocomplete on or off. Works with form and some types of input.
  • form - specifies the form the input belongs to.
  • formaction - specifies the URL of the file that will process the input when the form is submitted.
  • formenctype - specifies how the form-data should be encoded when submitted (only for forms with method="post")
  • formmethod - defines HTTP method for sending form-data to the action URL (overrides the form method). Works with input types sumbit and image.
  • formtarget - specifies a name or a keyword that indicates where to display the response that is received after submitting the form. (overrides the form target).
  • formnovalidate - specifies that in input element should not be validated when submitted. (overrides the form novalidate).
